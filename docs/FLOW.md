# Buyer Lite — Общий флоу проекта

## Описание

Система учёта для байера, работающего на Дордое (КР). Заменяет тетрадь и ручку, даёт прозрачность по клиентам, заказам, отправкам и финансам.

---

## Термины

| Термин | Описание |
|--------|----------|
| **Байер** | Пользователь системы, который закупает товары на Дордое и отправляет клиентам в Россию |
| **Клиент** | Человек из России, который отправляет деньги байеру и получает товары |
| **Точка (Поставщик)** | Место на Дордое, где закупается товар (Айлин, Дина и т.д.) |
| **Заказ** | Планируемая закупка у поставщика для конкретного клиента |
| **Отправка (Карго)** | Физическая отправка товаров клиенту через карго-службу |
| **Курс** | Ручной курс обмена RUB → KGS на момент операции |
| **Скрытая маржа (Курсовой спред)** | Разница между курсом, который байер называет клиенту, и реальным курсом обмена. Пример: клиенту говорят курс 1.16, а реально обменяли по 1.17 |
| **Товарная маржа** | Разница между ценой закупки у поставщика и ценой, которую байер называет клиенту. Пример: клиенту говорят цена 120 сом/шт, а реально купили за 115 сом/шт |
| **Комиссия байера** | Фиксированные 5% от цены товара (без учёта скрытой маржи) |

## Основной цикл (пошагово)

### Шаг 1: Клиент отправляет деньги

```
Клиент (Россия) → отправляет рубли → Байер получает
```

- Клиент отправляет деньги (обычно рубли RUB)
- Сумма **не всегда совпадает** с нужной — может быть больше или меньше
- На клиенте висит **баланс** (доступные средства)
- Основная валюта получения привязана к клиенту в настройках (по дефолту RUB)

**Поля клиента:**
- Имя
- Телефон / Telegram
- Основная валюта (по дефолту RUB, можно создать новую в настройках)
- Баланс (доступные средства)
- Адрес / город доставки (для карго)
- Комментарий

**Валюты:**
- По дефолту: RUB (рубль), KGS (сом)
- Можно создать новые в настройках (USD, EUR и т.д.)
- Валюты хранятся в таблице `currencies`

---

### Шаг 2: Байер обменивает валюту

```
RUB → KGS (ручной курс)
```

- Курс обмена фиксируется **вручную** (у каждого обменника свой курс)
- Байер может "играть на курсе": называет клиенту курс 1.16, а реально обменял по 1.17
- Разница — **курсовой спред** — идёт в пользу байера
- В отчёте клиенту отображается только курс, который байер назвал (без реального)

**Операция обмена:**
- Дата
- Клиент
- Сумма в валюте клиента (напр. 100 000 RUB)
- Реальный курс обмена (напр. 1.17)
- Курс для клиента (напр. 1.16)
- Итоговая сумма в KGS (для внутреннего учёта)
- Сумма в KGS для клиента (по его курсу)

---

### Шаг 3: Заказ у поставщика (планирование)

```
Байер планирует → формирует заказ на точке
```

- Байер идёт на конкретную точку (поставщик) на Дордое
- Выбирает товары, договаривается о цене
- Цена для клиента может отличаться от реальной (товарная маржа)

**Заказ:**
- Дата
- Точка (поставщик): Айлин, Дина и т.д.
- Клиент
- Список товаров (таблица):
  - ID товара (у поставщика)
  - Тип товара (блузка, юбка, платье, штаны и т.д.)
  - Количество
  - Цена закупки за штуку (реальная, в KGS)
  - Цена для клиента за штуку (может отличаться)
  - Необязательные поля: название, размер, цвет, фото
- Статус: запланирован / закуплен / готов к отправке

---

### Шаг 4: Приход товара

```
Поставщик → товар пришёл → байер проверяет
```

- Поставщик сообщает что товар готов
- Байер проверяет количество и качество
- Меняет статус заказа на "закуплен"

---

### Шаг 5: Отправка через Карго

```
Байер → фасует по клиентам → отправляет через карго
```

- Байер формирует **отправку** — группирует товары по клиентам
- На мешках пишет: тип товара, количество, куда едет, кому
- Выбирает какие товары на какого клиента отправляются

**Отправка:**
- Дата
- Список товаров (выбираются из закупленных заказов)
- Привязка к клиенту
- Куда едет (город/адрес)
- Комментарий ()
- Стоимость доставки (заполняется позже)

---

### Шаг 6: Оплата доставки на кассе

```
Байер → идёт на кассу → оплачивает доставку
```

- Байер заполняет стоимость доставки
- Эта сумма тоже списывается с баланса клиента

---

### Шаг 7: Формирование отчёта клиенту

```
Система → формирует PDF → байер отправляет клиенту
```

**Отчёт клиенту показывает:**
- Список заказанных товаров с ценами (ценами для клиента!)
- Количество каждого товара
- Стоимость доставки
- Итого (товары + доставка + комиссия 5%)
- **Не показывает:** реальный курс, реальную цену закупки, скрытую маржу

**Отчёт в формате PDF** — удобно для отправки в Telegram/WhatsApp

## Сущности и связи

```
Клиент (Client)
├── Баланс
├── Настройки (валюта, город)
├── Заказы (Orders)
│   ├── Поставщик (Supplier)
│   └── Товары (Items)
├── Отправки (Shipments)
│   └── Товары из заказов
├── Операции обмена валюты (ExchangeOperations)
└── Отчёты (Reports)

Байер (User)
├── Управляет клиентами
├── Создаёт заказы
├── Оформляет отправки
├── Фиксирует обмены
└── Генерирует отчёты
```

## Финансовая модель

### Баланс клиента

```
Баланс = (сумма полученных денег в KGS) 
       - (стоимость товаров по цене для клиента) 
       - (стоимость доставки) 
       - (комиссия 5%)
```

### Скрытая маржа байера

```
Курсовой спред = (реальный курс - курс для клиента) × сумма в валюте
Товарная маржа = (цена для клиента - цена закупки) × количество
Комиссия = сумма товаров для клиента × 5%
```

### Пример

Клиент отправил 100 000 RUB:
- Байер обменял по курсу 1.17 (реальный), клиенту сказал 1.16
- Курсовой спред: (1.17 - 1.16) × 100 000 = **1 000 KGS** в пользу байера
- Клиенту начислено: 100 000 × 1.16 = 116 000 KGS

Закуплено товаров:
- 100 блузок × 115 сом (реально) = 11 500 KGS
- Клиенту сказано: 100 × 120 = 12 000 KGS
- Товарная маржа: **500 KGS**

Доставка: 3 000 KGS
Комиссия: 12 000 × 5% = **600 KGS**

Итого списано с баланса клиента: 12 000 + 3 000 + 600 = 15 600 KGS
Остаток на балансе: 116 000 - 15 600 = 100 400 KGS

Общая маржа байера: 1 000 (курс) + 500 (товар) + 600 (комиссия) = **2 100 KGS**

## UI/UX принципы

1. **Простота** — байеру 52 года, минимум сложных элементов
2. **Мобильность** — PWA, Samsung, сохранение на главный экран
3. **Крупные кнопки** — удобно нажимать пальцем
4. **Минимум ввода** — где можно, использовать селекты вместо ручного ввода
5. **Понятные лейблы** — на русском языке
6. **Быстрый доступ** — главная страница: что сделать прямо сейчас?
7. **Тёмная тема** — не нужна, байер работает днём

## Отчётность (PDF)

- Генерация PDF на сервере
- Отображает только то, что клиент должен видеть
- Не содержит: реальный курс, реальную цену закупки, скрытую маржу
- Можно скачать или отправить (ссылка на PDF)
